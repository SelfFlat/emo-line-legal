# EMO LINE — публичные юридические документы

Публичное зеркало юридических страниц приложения **EMO LINE**, которые
обслуживаются через **GitHub Pages**. Исходный код приложения живёт в отдельном
приватном репозитории; сюда попадают только публичные документы.

## Страницы

| Путь | URL (после включения Pages) | Назначение |
|---|---|---|
| `index.html` | `/emo-line-legal/` | Указатель на документы (RU/EN) |
| `privacy/index.html` | `/emo-line-legal/privacy/` | Политика конфиденциальности (RU) |
| `privacy/en/index.html` | `/emo-line-legal/privacy/en/` | Privacy Policy (EN) |
| `terms/index.html` | `/emo-line-legal/terms/` | Условия использования + медицинская оговорка (RU) |
| `terms/en/index.html` | `/emo-line-legal/terms/en/` | Terms of Use + medical disclaimer (EN) |

Языки связаны переключателем RU ⇄ EN в шапке каждой страницы. **Держать локали
синхронными:** при изменении любого пункта править обе версии — RU и EN.

> **Статус: черновики.** На страницах стоят баннеры «Черновик», `noindex` и
> незаполненные поля `[…]`. Перед тем как ссылаться на них из приложения и
> подавать в Google Play Console — заполнить поля, снять баннеры и `noindex`,
> провести юридическую вычитку.

## GitHub Pages

- **Settings → Pages → Source:** *Deploy from a branch* → ветка `main`, папка **`/ (root)`**.
- `.nojekyll` отключает обработку Jekyll — HTML отдаётся как есть.
- URL после подачи в Play Console менять нельзя.

## Что пересмотреть при запуске синхронизации (v2)

Пункты в `terms/` (RU и EN) с бейджем **⟳ обновить при синхронизации** (§2 хранение,
§6 аккаунт) и парные разделы §1.3/§1.4 в `privacy/` завязаны на локальное хранение
и должны быть переписаны, когда появятся облачная синхронизация и авторизация
(включая отдельный URL удаления аккаунта, которого требует Google для приложений с входом).
Правки вносить синхронно в обе локали.
